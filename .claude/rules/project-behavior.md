# 项目整体行为规范

> 本文档定义全项目都需要遵守的通用行为准则，前端后端开发都适用。

---

## 代码规范

1. **优先复用**：优先复用已有组件和工具函数，不要重复实现相同功能。
2. **不随意重命名**：不要无理由重命名文件、函数或者目录。
3. **确认影响范围**：涉及公共接口或导出结构的修改，先确认影响范围，避免引入未知问题。
4. **保持风格一致**：新增代码与当前项目风格保持一致，包括命名规范、代码格式、注释等。
5. **少用 any**：项目中尽量少用 any 类型，优先定义具体的类型。

---

## 安全规范

1. 不直接操作生产环境配置
2. 不提交 API 密钥、Token 等敏感信息
3. 不泄露数据库连接信息、云服务密钥等
