# 前端安全检查清单

本文档定义了前端代码安全检查清单，用于安全漏洞扫描和代码审查。

---

## 🔴 高危漏洞

### 1. XSS 跨站脚本攻击

| 检查点 | 风险说明 |
|--------|----------|
| 是否直接将用户输入插入 `innerHTML` | 未转义用户输入可被利用执行任意 JavaScript |
| 是否在 `dangerouslySetInnerHTML` 使用用户输入 | 需要使用 DOMPurify 等库 sanitize |
| 是否使用用户输入拼接 `eval()` | `eval` 会直接执行代码，绝对禁止 |
| 是否使用 `document.write()` | 同上，存在 XSS 风险 |
| 是否使用用户输入设置 `href`/`src` 属性 | 可能构造 `javascript:` 链接执行代码 |
| 是否使用用户输入设置 `background-image` url | 可能构造 `url(javascript:...)` 执行代码 |
| 是否对用户输入进行了正确转义 | 输出到 HTML 必须转义特殊字符 |

### 2. 认证与授权

| 检查点 | 风险说明 |
|--------|----------|
| 需要认证的路由是否有保护 | 未保护的路由可直接绕过认证访问 |
| token 是否安全存储 | 项目使用 localStorage，需要检查是否正确清理 |
| token 过期是否正确处理 | 过期后必须清除并重定向登录 |
| 401/410 未授权是否正确跳转 | 需要清除所有 token，带 redirect 参数跳转 |
| 是否硬编码了 API keys / 密码 | 敏感信息不能提交到代码库 |
| refresh token 刷新逻辑是否正确 | 刷新失败必须清除所有 token |
| 是否存在权限绕过逻辑 | 前端权限控制只是 UX，后端必须二次校验 |

### 3. CSRF 跨站请求伪造

| 检查点 | 风险说明 |
|--------|----------|
| 状态改变请求（增删改）是否有 CSRF 保护 | 后端需要验证 CSRF token |

---

## 🟠 中高危漏洞

### 4. 敏感信息泄露

| 检查点 | 风险说明 |
|--------|----------|
| 是否在 `console.log` 打印敏感信息 | 生产环境用户可在控制台查看 |
| 是否在客户端存储密码 / 密钥 | 敏感数据不应存储在客户端 |
| 是否通过 URL 参数传递敏感信息 | URL 会被记录在日志、历史中 |
| 错误信息是否泄露服务器路径/架构 | 错误信息应该友好，不暴露内部细节 |
| 是否在 `localStorage` 存储多余敏感数据 | 只存必要信息，退出登录必须清理 |

### 5. 不安全依赖使用

| 检查点 | 风险说明 |
|--------|----------|
| 是否使用了已知有漏洞的依赖版本 | 需要定期 `npm audit` 检查 |
| 是否不必要使用 `dangerouslySetInnerHTML` | 能不用就不用，必须用要 sanitize |
| 是否引入了不再维护的过时包 | 替换为活跃维护的替代方案 |

---

## 🟠 中危漏洞

### 6. 点击劫持

| 检查点 | 风险说明 |
|--------|----------|
| 是否需要防止被 iframe 嵌入 | 登录页等敏感页面需要设置 CSP |

### 7. 不安全跳转

| 检查点 | 风险说明 |
|--------|----------|
| 使用用户可控路径跳转是否验证 | 用户可控的 redirect 参数需要验证白名单 |
| 是否允许 `javascript:` 链接 | 绝对禁止，会导致 XSS |
| `target="_blank"` 是否加 `rel="noopener noreferrer"` | 防止新窗口反向操纵原页面 |

### 8. 表单安全

| 检查点 | 风险说明 |
|--------|----------|
| 密码输入是否设置 `type="password"` | 否则会被浏览器记住并显示 |
| 文件上传是否验证文件类型 | 限制允许的 MIME 类型和扩展名 |
| 用户上传的图片是否允许 SVG | SVG 可能包含 XSS 代码 |

---

## 🟡 低危问题

### 9. 正则表达式注入

| 检查点 | 风险说明 |
|--------|----------|
| 用户输入是否直接用于正则表达式 | 可能导致 ReDoS 拒绝服务攻击 |

### 10. 混合内容

| 检查点 | 风险说明 |
|--------|----------|
| 是否在 HTTPS 页面加载 HTTP 资源 | 混合内容会被浏览器拦截，也不安全 |

---

## 🔵 信息提示（最佳实践）

| 检查点 | 建议 |
|--------|------|
| 是否使用 HTTPS 生产环境 | 必须 |
| 敏感 cookie 是否设置 HttpOnly | HttpOnly 防止 XSS 窃取（后端配置） |
| 是否设置 CSP 内容安全政策 | 推荐开启，进一步防止 XSS |

---

## 风险等级定义

| 等级 | 图标 | 说明 | 需要立即处理 |
|------|------|------|-------------|
| 高危 | 🔴 | 可被利用获取用户信息、权限绕过、XSS 攻击 | ✅ 是 |
| 中危 | 🟠 | 存在潜在安全风险，需要修复 | 尽快修复 |
| 低危 | 🟡 | 需要关注，但不紧急 | 有空修复 |
| 信息 | 🔵 | 信息提示，建议改进 | 可选 |

---

## 项目特有检查点（React H5 移动端）

这些是本项目需要特别关注的：

1. **Token 清理**：退出登录 / token 过期必须同时清除 `accessToken` 和 `refreshToken`
2. **路由保护**：需要登录的页面必须在 `RouteInterceptor` 中添加认证检查
3. **文章内容渲染**：用户生成内容渲染必须检查是否正确 sanitize 后才能使用 `dangerouslySetInnerHTML`
4. **分享链接跳转**：外部链接跳转必须验证，不能直接跳用户可控 URL

## 运行安全审计

使用以下命令运行安全审计：
```bash
# 依赖漏洞检查
npm audit

# 运行 AI 安全代码审计
/security src/
```
