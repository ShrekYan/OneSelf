# 项目整体概览生成规范

当需要快速了解整个项目全貌时，按照本规范自动探索并生成项目全景报告，帮助开发者快速上手。

**使用场景**:
- 新入职项目，需要快速了解整体情况
- 接手陌生项目，需要一份结构化的项目导览
- 跨团队协作，需要快速掌握项目结构

---

## 核心原则

1. **全面不啰嗦**: 覆盖整体架构，但不深入细节
2. **结构化输出**: 使用表格、列表、树形结构，便于快速扫描
3. **识别技术栈**: 自动识别前后端技术栈和版本
4. **实用导向**: 重点突出「从哪里开始」「怎么运行」
5. **适配全栈**: 同时支持前端和后端，当前项目是全栈分离架构

---

## 分析步骤

1. **探索根目录结构**
   - 识别是否有前后端分离（`src/` + `backend/`）
   - 识别关键配置文件位置

2. **收集技术栈信息**
   - 读取根目录 `package.json` → 前端依赖版本
   - 如果有后端，读取 `backend/package.json` → 后端依赖版本
   - 读取 `CLAUDE.md` → 项目开发规范要点

3. **梳理目录结构**
   - 按目录分类，说明每个主要目录的用途
   - 识别核心业务模块位置

4. **识别核心功能**
   - 根据目录和路由，推导项目提供的核心功能
   - 按业务领域整理功能模块

5. **整理开发指南**
   - 提取关键开发命令（安装、启动、构建、检查）
   - 记录默认访问地址

6. **输出结构化报告**
   - 按照下面的模板输出
   - 保持精简，拒绝水文

## 开发指南

### 前端开发命令

```bash
npm install          # 安装依赖
npm run dev          # 启动开发服务
npm run build        # 生产构建
npm run lint         # 代码检查
npx tsc --noEmit     # 类型检查
```

**开发地址**: http://localhost:xxxx

### 后端开发命令（如存在）

```bash
cd backend
npm install          # 安装依赖
npm run start:dev    # 启动开发服务
npm run build        # 生产构建
```

**API 地址**: http://localhost:xxxx/api/
**文档地址**: http://localhost:xxxx/docs
