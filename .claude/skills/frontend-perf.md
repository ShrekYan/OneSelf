# 性能优化审查指南

当需要分析和优化前端性能时，按照本指南进行系统性检查，聚焦移动端 H5 特定性能问题。

---

## 性能问题分类

| 类别 | 常见症状 |
|------|----------|
| **加载性能** | 白屏时间长、首屏加载慢、资源体积大 |
| **交互性能** | 滚动卡顿、点击响应慢、动画掉帧 |
| **内存性能** | 随着使用变慢、页面后退不释放、泄漏 |
| **网络性能** | 请求过多、重复请求、缓存无效 |

---

## 检查清单（按优先级）

### 🔴 高优先级 - 严重影响用户体验

#### 1. React 重渲染问题
- [ ] 是否存在对象/函数字面量直接传入 props 导致每次重渲染？
- [ ] `useObserver` 是否包裹在正确位置，避免过度订阅？
- [ ] 大组件是否未拆分，状态变化导致整棵树重渲染？
- [ ] 列表渲染是否使用了不稳定的 `key`（如 `index`、随机值）？

#### 2. MobX 状态优化
- [ ] 状态粒度是否合理？是否把不相关的字段放在同一个可观察对象中？
- [ ] 计算属性是否使用 `computed` 缓存？
- [ ] 是否在 render 中进行昂贵计算？
- [ ] 页面退出后状态是否清理？

#### 3. 长列表性能
- [ ] 列表超过 50 项是否使用虚拟滚动？
- [ ] 是否一次性渲染所有数据？
- [ ] 是否分页/无限滚动加载？

#### 4. 图片资源优化
- [ ] 非首屏图片是否懒加载？
- [ ] 大图是否压缩？是否使用 WebP 格式？
- [ ] 图片是否设置了宽高比，避免 CLS 布局偏移？
- [ ] 是否有大图小用（用 2x 图展示在 1x 屏幕）？

#### 5. 内存泄漏
- [ ] `setInterval`/`setTimeout` 是否在 `useEffect` 清理？
- [ ] `addEventListener` 是否移除？
- [ ] 全局缓存/Map 是否无限制增长？
- [ ] 路由缓存是否缓存了过多页面？

### 🟠 中优先级 - 影响性能体验

#### 打包体积
- [ ] 是否整体引入大库（如 `react-use`、`lodash`）而不是按需引入？
- [ ] 是否引入了未使用的第三方依赖？
- [ ] Ant Design Mobile 是否按需加载？

#### 网络请求
- [ ] 不常变数据是否启用 API 缓存（`cache: true`）？
- [ ] 是否有重复并发请求（未被 `cancel-manager` 取消）？
- [ ] 是否可以合并多个小请求？
- [ ] 是否需要时才请求，而非预加载过多数据？

#### 样式性能
- [ ] SCSS 嵌套是否超过 3 层？
- [ ] 是否过度使用昂贵 CSS 属性（`box-shadow`、`filter`）？
- [ ] 动画是否开启硬件加速？

#### 移动端交互
- [ ] 频繁触发的事件（scroll、input、resize）是否防抖节流？
- [ ] 可点击区域是否 ≥ 44px × 44px？
- [ ] 是否适配安全区域？

### 🟡 低优先级 - 进一步优化

- [ ] 是否利用 React 19 特性（`useTransition`、`useDeferredValue`）处理非紧急更新？
- [ ] 非首屏路由是否懒加载？
- [ ] 大量数据计算是否放到 Web Worker？
- [ ] 关键资源是否预加载？

---

## 项目已内置优化设施，检查是否正确使用

| 优化设施 | 位置 | 使用场景 | 正确用法 |
|----------|------|----------|----------|
| **请求缓存** | `src/api/request-cache.ts` | 不常变的 GET 数据 | `api.get(url, { cache: true, cacheTime: 5 * 60 * 1000 })` |
| **重复请求取消** | `src/api/cancel-manager.ts` | 并发重复请求 | 自动处理，无需手动 |
| **路由缓存** | `react-activation` + `react-freeze` | 保留列表状态 | 只缓存列表页，不缓存详情页 |
| **自动重试** | `axios-instance.ts` | 网络抖动失败 | 自动重试幂等请求 |

## 问题严重程度分级

| 级别 | 说明 |
|------|------|
| 🔴 **严重** | 明显卡顿、内存泄漏、功能不可用、白屏时间过长 |
| 🟠 **中等** | 影响体验、可感知变慢、但功能可用 |
| 🟡 **轻微** | 潜在优化点、不影响当前使用 |
