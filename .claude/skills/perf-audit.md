# 前端性能审计规范 (perf-audit)

本文档定义了对项目代码进行 Web Vitals 性能审计的检查规则，专注于三大核心指标：**LCP (最大内容绘制)**、**FID (首次输入延迟)**、**CLS (累积布局偏移)**。

---

## 概述

当用户请求对代码进行性能检查时，按照本规范：
1. 读取受检代码文件
2. 按照 LCP/FID/CLS 三个维度逐一检查
3. 识别潜在问题点
4. 给出具体可落地的优化建议

---

## 一、LCP (Largest Contentful Paint) - 最大内容绘制检查

### LCP 性能目标
- 优良：≤ 2.5s
- 需要改进：2.5s ~ 4.0s
- 差：> 4.0s

### 常见问题检查清单

#### 图像相关问题

- [ ] **LCP 元素是大图且未优化**
  - 检查：首屏大图未压缩、体积过大
  - 问题：直接下载大图会阻塞 LCP
  - 优化建议：
    - 使用现代图片格式（WebP/AVIF）
    - 对 LCP 图片使用 `width`/`height` 属性设置正确尺寸
    - 使用 `loading="eager"` 让 LCP 图片优先加载
    - 配置 CDN 自适应分辨率

- [ ] **缺少图片懒配置**
  - 检查：非首屏图片没有 `loading="lazy"`
  - 优化建议：非首屏图片添加懒加载，释放带宽给 LCP

- [ ] **LCP 图片被 CSS 背景图实现**
  - 检查：首屏大Banner使用 `background-image` 而非 `<img>`
  - 问题：CSS 背景图预加载扫描器难以提前发现，会延迟加载
  - 优化建议：改用 `<img>` 标签，让浏览器提前发现并预加载

- [ ] **缺少预加载**
  - 检查：关键图片/字体没有 `<link rel="preload">`
  - 优化建议：对 LCP 图片添加预加载

#### 资源阻塞问题

- [ ] **未切割大体积 JS/CSS**
  - 检查：入口 bundle 体积过大（> 200KB gzipped）
  - 问题：阻塞渲染进程，延迟 LCP
  - 优化建议：
    - 使用路由懒加载代码分割
    - 第三方库拆包分离
    - 异步加载非首屏组件

- [ ] **渲染阻塞 CSS 未优化**
  - 检查：大量 CSS 放在 `<head>` 中阻塞渲染
  - 优化建议：
    - 非关键 CSS 拆分异步加载
    - 内联首屏关键 CSS
    - 使用 `media="print"` 等方式让非匹配 CSS 不阻塞

- [ ] **第三方脚本阻塞**
  - 检查：统计、广告、分享等第三方脚本放在头部同步加载
  - 优化建议：
    - 添加 `async`/`defer` 属性
    - 延迟加载到 LCP 之后
    - 使用 `preconnect` 提前建立连接

#### 服务器问题

- [ ] **缺少 CDN 加速**
- [ ] **未开启 gzip/Brotli 压缩**
- [ ] **HTTP 慢连接**

## 二、FID (First Input Delay) - 首次输入延迟检查

### FID 性能目标
- 优良：≤ 100ms
- 需要改进：100ms ~ 300ms
- 差：> 300ms

> 注意：现在 FID 正被 INP (Interaction to Next Paint) 替代，但检查原则类似。

### 常见问题检查清单

- [ ] **长任务阻塞主线程**
  - 检查：
    - 大体积框架初始化在主线程执行
    - 大量循环计算在主线程
    - 复杂组件首次渲染耗时过长
  - 优化建议：
    - 使用 `requestIdleCallback` 推迟非紧急任务
    - 将大计算拆分到多个 `requestAnimationFrame` 批次执行
    - 使用 Web Worker 处理复杂计算

- [ ] **过多同步 DOM 操作**
  - 检查：初始化阶段多次连续修改 DOM
  - 优化建议：
    - 使用文档片段批量更新
    - 先 `display: none` 更新完再显示

- [ ] **事件监听器未优化**
  - 检查：`scroll`/`mousemove`/`resize` 事件没有防抖/节流
  - 优化建议：使用 `debounce`/`throttle` 限制执行频率

- [ ] **第三方脚本占用主线程**
  - 检查：广告、分析、iframe 等第三方脚本大量占用主线程时间
  - 优化建议：
    - 懒加载第三方组件
    - 移除不必要的第三方脚本

- [ ] **大量 JS 在加载时执行**
  - 检查：`DOMContentLoaded`/`load` 事件中执行大量任务
  - 优化建议：拆分任务，推迟非关键初始化

## 三、CLS (Cumulative Layout Shift) - 累积布局偏移检查

### CLS 性能目标
- 优良：≤ 0.1s
- 需要改进：0.1s ~ 0.25s
- 差：> 0.25s

### 常见问题检查清单

- [ ] **图片缺少宽高属性**
  - 检查：`<img>` 没有设置 `width` 和 `height` HTML 属性
  - 问题：浏览器无法在加载前预留空间，图片加载后会挤开内容
  - 优化建议：
    - 始终设置 `width` 和 `height` 属性
    - 使用 CSS `aspect-ratio` 保持比例
    - 现在 CSS 已经可以自动根据属性计算比例

- [ ] **动态插入内容没有预留空间**
  - 检查：广告、弹窗、消息提示动态插入顶部，推挤下方内容
  - 优化建议：
    - 提前为动态内容预留空间
    - 将动态内容放在视口下方，不影响现有内容布局

- [ ] **字体加载导致文字闪动**
  - 检查：自定义字体加载慢，文字一开始不可见或显示 fallback 字体后替换
  - 优化建议：
    - 使用 `font-display: swap`
    - 预加载关键字体
    - 使用系统字体作为 fallback，减少跳动幅度

- [ ] **弹窗/动画未从正确位置展开**
  - 检查：弹窗从上方滑入过程中持续推动内容产生偏移
  - 优化建议：弹窗使用 `position: fixed/fixed` 脱离文档流

- [ ] **响应式图片未使用 srcset**
  - 检查：固定尺寸大图在小屏加载后缩放，造成布局重排
  - 优化建议：使用 `srcset` + `sizes` 让浏览器选择合适尺寸

- [ ] **没有使用 contain/intrinsic 尺寸容器**
  - 检查：iframe、embed、video 没有设置尺寸
  - 优化建议：
    ```css
    .video-container {
      position: relative;
      height: 0;
      padding-bottom: 56.25%; /* 16:9 */
    }
    iframe {
      position: absolute;
      top: 0;
      left: 0;
      width: 100%;
      height: 100%;
    }
    ```

## 四、本项目特定检查点

基于当前 React + Vite + H5 项目，额外检查：

### React 特定

- [ ] **不必要的全量重渲染**
  - 检查：父组件更新导致所有子组件重渲染
  - 优化建议：使用 `React.memo` + `useMemo` + `useCallback` 缓存

- [ ] **组件懒加载未使用**
  - 检查：大型页面/组件没有使用 `React.lazy` 懒加载
  - 优化建议：路由级别使用 lazy + Suspense

### MobX 特定

- [ ] **过度使用 `useObserver`**
  - 检查：所有组件都包 observer，导致不必要的重渲染
  - 优化建议：只在使用了 observable 状态的组件包 observer

### 移动端特定

- [ ] **首屏请求过多**
  - 检查：首屏发起超过 10 个以上网络请求
  - 优化建议：合并小资源，减少请求数

- [ ] **未适配低速网络**
  - 检查：没有骨架屏/占位，用户长时间看到空白
  - 优化建议：添加骨架屏占位，改善感知性能

## 六、检查工作流

1. **确定范围**: 用户要求检查哪些文件
2. **读取代码**: 逐文件分析
3. **对照检查清单**: 逐一排查
4. **分类问题**: 按 LCP/FID/CLS 分类
5. **给出建议**: 每个问题配具体代码修改示例
6. **排序优先级**: 告诉用户先改什么

## 示例审计输出

### ❌ 发现问题 - 示例

#### CLS 问题
- **问题位置**: `src/pages/Home/components/Banner.tsx:15` - 首屏 Banner 图片没有设置 width 和 height 属性
- **影响**: 图片加载完成后会向下推挤内容，产生较大布局偏移，CLS 分数升高
- **优化建议**:
  ```tsx
  // 修改前
  <img src={bannerUrl} className={styles.banner} />

  // 修改后
  <img
    src={bannerUrl}
    width={750}
    height={360}
    className={styles.banner}
    loading="eager"
  />
  // CSS 会自动根据属性计算 aspect-ratio，预留空间
  ```

#### LCP 问题
- **问题位置**: `src/main.tsx:8` - 百度统计脚本同步加载阻塞渲染
- **影响**: 脚本下载执行期间阻塞主线程，延迟 LCP
- **优化建议**:
  ```html
  <!-- 添加 async 延迟执行 -->
  <script async src="https://hm.baidu.com/hm.js?xxx"></script>
  ```
