# Backend 项目开发指南

本文件是 **backend** 项目的开发规范，Claude Code 协助开发时请严格遵循此处定义的规则。

---

## 📋 项目概览

**项目描述**: 博客系统后端 API 服务，为前端 H5 应用提供数据接口支持。

**技术栈**: NestJS 11 + TypeScript 5.7 + Node.js 20+

**架构风格**: NestJS 模块化分层架构，遵循领域驱动设计，按业务领域划分模块。

**核心设计理念**:
- 模块化：每个业务领域独立成模块
- 分层职责：Controller 处理 HTTP，Service 处理业务逻辑
- 依赖注入：充分利用 NestJS 依赖注入系统
- 类型安全：全程 TypeScript 严格模式
- API 优先：自动生成 Swagger 文档

---

## 📁 项目结构

```
backend/
├── src/                          # 源代码目录
│   ├── [module]/                 # 业务模块（按领域划分，小写命名）
│   │   ├── dto/                  # 数据传输对象（请求/响应）
│   │   ├── *.controller.ts       # 控制器（处理 HTTP 请求）
│   │   ├── *.service.ts          # 服务（业务逻辑实现）
│   │   └── *.module.ts           # 模块定义
│   ├── common/                   # 公共基础设施模块
│   │   ├── decorators/           # 自定义装饰器
│   │   ├── dto/                  # 公共 DTO 基类
│   │   ├── filters/              # 全局异常过滤器
│   │   └── interceptors/         # 全局拦截器
│   ├── app.controller.ts         # 根控制器
│   ├── app.module.ts             # 根模块
│   ├── app.service.ts            # 根服务
│   └── main.ts                   # 应用入口
├── test/                         # e2e 测试
├── dist/                         # 编译输出（git 忽略）
├── .env                          # 默认环境配置
├── .env.development              # 开发环境配置
├── .env.production               # 生产环境配置
├── .prettierrc                   # Prettier 配置
├── .eslint.config.mjs            # ESLint 配置
├── nest-cli.json                 # NestJS CLI 配置
├── package.json                  # 依赖声明
├── tsconfig.json                 # TypeScript 配置
└── tsconfig.build.json           # TypeScript 构建配置
```

### 命名约定

| 元素 | 命名规则 | 示例 |
|------|----------|------|
| 模块目录 | 小写，按业务领域 | `auth`, `category`, `article` |
| 文件名 | kebab-case + 后缀 | `auth.controller.ts`, `login.dto.ts` |
| 类名 | PascalCase | `AuthController`, `ArticleService` |
| 变量/函数 | camelCase | `getUserById`, `userId` |
| DTO 类 | PascalCase + 后缀 | `LoginDto`, `LoginResponseDto` |

## ⚡ 常用开发命令

```bash
# 安装依赖
npm install

# 开发模式启动（热重载）
npm run start:dev

# 调试模式启动
npm run start:debug

# 构建生产版本
npm run build

# 生产环境启动
npm run start:prod

# ESLint 检查并修复
npm run lint

# Prettier 格式化所有代码
npm run format

# 查看帮助
nest --help
```

## ✅ 验证流程

开发完成后，请务必依次执行以下检查:

1. **代码检查**: `npm run lint` → 修复所有错误和警告
2. **代码格式化**: `npm run format`
3. **单元测试**: `npm run test` → 确保所有测试通过
4. **构建检查**: `npm run build` → 确保能正常编译
5. **手动验证**: 启动服务 `npm run start:dev`，验证接口功能正常
