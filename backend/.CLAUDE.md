# Backend 项目开发指南

本文件是 **backend** 项目的开发规范，Claude Code 协助开发时请严格遵循此处定义的规则。

---

## 📋 项目概览

**项目描述**: 博客系统后端 API 服务，为前端 H5 应用提供数据接口支持。

**技术栈**: NestJS 11 + TypeScript 5.7 + Node.js 20+

**架构风格**: NestJS 模块化分层架构，遵循领域驱动设计，按业务领域划分模块。

**核心设计理念**:
- 模块化：每个业务领域独立成模块
- 分层职责：Controller 处理 HTTP，Service 处理业务逻辑
- 依赖注入：充分利用 NestJS 依赖注入系统
- 类型安全：全程 TypeScript 严格模式
- API 优先：自动生成 Swagger 文档

## 🎯 开发规范

### 1. 架构原则

- **按业务领域划分模块**: 每个核心业务功能独立成一个模块
- **分层职责分离**:
  - **Controller 层**: 只负责 HTTP 请求处理、路由绑定、参数绑定、API 文档装饰，**不包含业务逻辑**
  - **Service 层**: 负责实现所有业务逻辑，保持纯净可测试
  - **DTO 层**: 定义请求/响应的数据结构和验证规则
  - **Module 层**: 组织依赖，声明控制器和服务
- **依赖注入**: 使用构造函数注入，不要使用属性注入
- **单一职责**: 一个类只负责一件事

### 2. DTO 规范

**强制要求**: 所有请求入参和响应数据必须使用 DTO 类定义。

```typescript
// ✅ 正确示例
import { IsString, IsEmail, MinLength } from 'class-validator';
import { ApiProperty } from '@nestjs/swagger';

export class LoginDto {
  @ApiProperty({ description: '用户名', example: 'admin' })
  @IsString()
  username: string;

  @ApiProperty({ description: '密码，至少6位', example: '123456' })
  @IsString()
  @MinLength(6)
  password: string;
}
```

规则:
- 每个 DTO 属性必须添加 `@ApiProperty` 装饰器（Swagger 文档）
- 每个 DTO 属性必须添加 `class-validator` 验证装饰器
- 响应数据也使用 DTO 包装，并添加 Swagger 文档
- 使用 `whitelist: true` 全局配置自动过滤多余字段

### 3. 认证授权规范

- 使用 JWT 进行身份认证
- 路由默认需要认证，使用 `@Public()` 装饰器标记公开接口
- 在 Controller 方法参数中使用 `@CurrentUserId()` 获取当前登录用户 ID
- 敏感路由使用 `AuthGuard` 保护

```typescript
// 公开接口示例
@Public()
@Post('login')
async login(@Body() loginDto: LoginDto) {
  return this.authService.login(loginDto);
}

// 需要认证接口示例
@Post('logout')
async logout(@CurrentUserId() userId: number) {
  return this.authService.logout(userId);
}
```

### 4. Controller 规范

```typescript
@ApiTags('认证')  // ✅ 必须添加模块标签
@Controller('auth') // ✅ 路由前缀使用小写
export class AuthController {
  constructor(private readonly authService: AuthService) {} // ✅ 只读注入

  @Post('login') // ✅ 路由使用小写
  @ApiOperation({ summary: '用户登录' }) // ✅ 必须添加接口说明
  @ApiBody({ type: LoginDto }) // ✅ 声明请求体类型
  @ApiResponse({ status: 200, description: '登录成功', type: LoginResponseDto }) // ✅ 声明响应
  @ApiResponse({ status: 401, description: '用户名或密码错误' })
  async login(
    @Body() loginDto: LoginDto,
  ): Promise<LoginResponseDto> { // ✅ 必须声明返回类型
    return this.authService.login(loginDto);
  }
}
```

规则:
- 每个 Controller 必须添加 `@ApiTags()` 分类标签
- 每个接口必须添加 `@ApiOperation()` 说明用途
- 每个接口必须添加 `@ApiResponse()` 说明不同响应情况
- 必须声明方法返回类型 Promise<ResponseDto>

### 5. Service 规范

- Service 只包含业务逻辑，不包含 HTTP 处理代码
- 使用依赖注入方式，保持可测试性
- 方法命名遵循 `get*`, `list*`, `create*`, `update*`, `delete*` 规则
- 必须为方法参数和返回值添加类型声明

### 6. Module 规范

- 每个业务模块必须有一个 `.module.ts` 文件
- Controller 和 Provider 声明在对应模块中
- 需要被其他模块使用的 Service 需要 `exports`
- 根模块 `AppModule` 导入所有业务模块

### 7. API 文档规范

**所有接口必须完整添加 Swagger 装饰器**:

| 装饰器 | 使用场景 |
|--------|----------|
| `@ApiTags()` | Controller 级别，分组标签 |
| `@ApiOperation()` | 接口级别，说明接口用途 |
| `@ApiProperty()` | DTO 属性，说明字段含义和示例 |
| `@ApiResponse()` | 接口级别，说明各种响应情况 |
| `@ApiBody()` | POST/PUT 接口，声明请求体类型 |

### 8. 错误处理规范

- 使用 `HttpException` 或其子类抛出异常
- 不要吞掉异常，让全局异常过滤器统一处理
- 正确使用 HTTP 状态码:
  - `200`: 成功
  - `400`: 请求参数错误
  - `401`: 未认证
  - `403`: 无权限
  - `404`: 资源不存在
  - `500`: 服务器错误

```typescript
// ✅ 正确示例
if (!user) {
  throw new HttpException('用户名或密码错误', HttpStatus.UNAUTHORIZED);
}
```

### 9. 环境配置规范

- 使用 `ConfigService` 读取环境变量，不要硬编码
- 不同环境使用不同配置文件: `.env.development`, `.env.production`
- 敏感信息（密钥、密码）只在环境变量中配置，不提交到代码库
- `ConfigModule` 已经配置为全局模块，可以直接注入使用

```typescript
constructor(private configService: ConfigService) {
  const jwtSecret = this.configService.get<string>('JWT_SECRET');
}
```

### 10. TypeScript 规范

- 开启严格模式 (`strict: true`)，必须遵守
- **禁止使用 `any` 类型**，使用 `unknown` 代替
- 必须为所有函数参数和返回值添加显式类型声明
- 优先使用 `interface` 定义对象结构，使用 `type` 定义联合类型和工具类型
- 充分利用类型推断，局部简单变量不需要显式声明
- 错误捕获必须进行类型收窄:

```typescript
try {
  await someAsyncOperation();
} catch (error) {
  if (error instanceof Error) {
    logger.error(error.message);
  }
}
```

## 📝 导入排序规范

导入按以下顺序分组，每组之间空一行:

1. **NestJS 框架包** (`@nestjs/*`)
2. **第三方依赖包** (class-validator, rxjs 等)
3. **内部模块导入** (./, ../)

```typescript
// ✅ 正确排序示例
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';

import { AuthController } from './auth.controller';
import { AuthService } from './auth.service';
```

- 密码必须使用 bcrypt 加密存储，不存储明文
- JWT 密钥必须从环境变量读取，不硬编码
- 所有用户输入必须经过 `class-validator` 验证
- CORS 已经全局配置，允许跨域访问
- 启用 CORS 允许所有来源，开发环境不需要限制
