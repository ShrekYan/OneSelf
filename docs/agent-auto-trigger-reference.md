## 🎯 触发机制概述

### 什么是平台级自动触发？

Claude Code Agent 的 `triggers` 配置是平台原生支持的自动路由机制。当用户输入匹配到某个 Agent 的触发词时，平台会直接将对话路由到该 Agent，跳过主 Agent 的判断流程。

### 工作原理

```
用户输入文本
    ↓
Claude Code 平台层扫描所有 Agent 的 frontmatter
    ↓
匹配到对应 Agent 的 triggers 列表
    ↓
直接进入该 Agent 的上下文环境
    ↓
✅ 真正的自动调用，零人工判断
```

### vs "伪自动"方案对比

| 维度         | 平台级 triggers                    | 主 Agent 读文档判断                   |
| ------------ | ---------------------------------- | ------------------------------------- |
| **可靠性**   | 100% 确定性匹配                    | 依赖大模型理解，可能遗漏              |
| **响应速度** | 毫秒级匹配，立即路由               | 需要先读文档 + 逻辑判断（慢 5-10 倍） |
| **维护成本** | 每个 Agent 自描述触发条件          | 集中维护文档表格 + 指望主 Agent 理解  |
| **职责边界** | 清晰：平台负责路由，Agent 负责业务 | 主 Agent 兼任"调度员"角色             |
| **可扩展性** | 新增 Agent 只需加自己的 triggers   | 需要同步更新文档表格                  |

## 🔍 常见问题排查

### Q1：为什么我输入了触发词但没有自动进入 Agent？

**可能原因**：

1. 触发词不完全匹配（平台是精确匹配还是模糊匹配需要确认）
2. 触发词在 Agent 配置中缺失
3. Agent 文件位置不对（需要在 `.claude/agents/` 目录下）

**解决方法**：

- 检查 Agent 配置文件中的 triggers 列表
- 尝试使用最精确的触发词（如用完整的"开发前端页面"而非"前端页面"）
- 查看 Agent 文件是否在正确的目录

---

### Q2：多个 Agent 触发词重叠怎么办？

**现象**：用户输入可能同时匹配多个 Agent

**解决方法**：

1. 优化触发词，让每个 Agent 的触发词更精确
   - 不要用"开发"，要用"开发前端页面"或"开发后端接口"
2. 为高频触发词添加权重（如果平台支持）
3. 在 Agent 描述中明确区分职责范围

---

### Q3：如何验证我的 triggers 配置是否生效？

**验证步骤**：

1. 检查 Agent 文件的 YAML frontmatter 语法正确
2. 确认 `triggers` 字段缩进正确（2个空格）
3. 确认触发词是数组格式，每项前面加 `- `
4. 重启 Claude Code 或重新加载项目配置
5. 输入触发词测试是否自动路由

---

### Q4：triggers 支持正则表达式吗？

**当前状态**：根据 Claude Code 官方文档，目前 triggers 是精确字符串匹配。

**未来可能**：如果平台支持正则，将可以实现更灵活的匹配。
